# 🐢 리처드 대니스 원칙 기반 코스피(KOSPI) 투자 가이드 (종목 추가/삭제 기능 포함)

- 본 가이드는 투자 자산의 1%를 단일 매매 최대 허용 손실(1유닛 리스크)로 설정하는 리처드 대니스의 오리지널 규칙을 기반으로 합니다.
- 시스템의 지속 가능성을 위해 **유동적 종목 편집(추가/삭제) 프로토콜**을 내장하였습니다.

---

## 🛠️ [기능] 포트폴리오 종목 관리 프로토콜

터틀 시스템은 시장 주도주 변화에 따라 관리 종목을 유연하게 교체해야 합니다. 아래 규칙에 따라 종목을 추가하거나 삭제하십시오.

### ➕ 종목 추가 조건 (Add Protocol)

새로운 종목을 시스템에 편입할 때는 HTS/MTS를 켜고 아래 3가지를 검증한 후 테이블에 추가합니다.

1. **유동성 검증**: 코스피 200 이내 또는 시가총액 100위 이내 대형주일 것 (체결 오차 방지).
2. **변동성 측정**: 최근 20일 이동평균 ATR(N값)을 확인하여 1유닛 수량을 먼저 계산할 것.
3. **추세 필터**: 주가가 역배열 상태가 아닌, 60일 또는 120일 이동평균선 위에 위치해 있을 것.

### ❌ 종목 삭제 조건 (Delete Protocol)

아래 조건 중 하나라도 해당하면 즉시 관심 종목 테이블에서 삭제하고 감시를 중단합니다.

1. **역추세 고착화**: 주가가 20일 고점을 수개월간 돌파하지 못하고 거래량이 전멸한 경우.
2. **기업 펀더멘털 훼손**: 관리종목 지정, 횡령, 배임 등 리스크 관리 범위를 벗어난 대외 악재 발생 시.
3. **투자 자산 재배치**: 더 강력한 추세를 보이는 신규 추가 종목이 있어 유닛 한도가 부족할 경우.

## 🚦 2. 실전 매매 및 종목 순환 프로세스

### 1단계: 돌파 모니터링 및 종목 필터링

- 등록된 종목 중 **최근 20거래일 최고가**를 돌파하는 종목이 나오면 해당 종목의 '1유닛 수량'만큼 1차 매수합니다.
- 20일 돌파 신호가 장기간 나오지 않고 우하향하는 종목은 과감히 **삭제 프로토콜**을 가동해 포트폴리오에서 제외합니다.

### 2단계: 피라미딩 (불타기 추가 매수)

- 주가가 최초 진입 가격보다 **`+0.5 ATR` 만큼 상승**할 때마다 1유닛씩 추가 매수합니다. (최대 4유닛 제한)
- 포지션을 보유 중인 종목이 있을 때는 전체 투자 자산 중 사용 가능한 리스크 한도를 고려하여 신규 종목 추가를 제한합니다.

### 3단계: 청산 및 종목 교체

- **손절**: 주가가 **`-2 ATR`**까지 추락하면 기계적으로 전량 손절합니다.
- **익절**: 주가가 **최근 10거래일 최저가**를 깰 때 전량 매도하여 수익을 확정합니다.
- 청산이 완료된 종목 자리는 시장 흐름에 맞춰 **추가 조건**을 충족하는 새로운 코스피 대형주로 교체(순환매)합니다.

## 🔍 매수 후보 스캔 프로토콜

기존 8종목 외에 시스템 진입 조건을 충족하는 **신규 KOSPI 대형주를 자동 추출**합니다. 매 대시보드 실행마다 함께 갱신되며, 결과는 대시보드 "매수 후보" 영역에 등급별로 표시됩니다.

### 후보군 (Candidate Universe)

- **KOSPI 시가총액 상위 ~30종목** (보유 8종목 제외)
- 코스피 200 또는 시총 100위 이내라는 추가 프로토콜 1항(유동성) 충족
- 정확한 종목 리스트는 `src/scan.ts` 의 `CANDIDATES` 배열로 관리

### 평가 조건 (4개 모두 충족 시 A등급)

1. **종가 ≥ 20일 신고가** — System 1 entry, 추세 시작 신호
2. **종가 > MA60** — 중기 추세 위
3. **종가 > MA120** — 장기 추세 위
4. **MA60 > MA120** — 정배열 (추세 일치)

### 출력 등급

- **A등급 (즉시 진입 신호)**: 4개 조건 전부 충족. 시스템 1유닛만큼 즉시 매수 검토 가능
- **B등급 (돌파 임박)**: 추세 조건(2,3,4) 충족 + 20일 고점까지 **3% 이내**. 관찰 리스트
- **그 외**: 후보 제외

### 자본 한도 고려

신호가 떴다고 무조건 진입하지 않습니다. **1유닛 사이즈 = (총자산 × 1%) ÷ (2 × ATR)** 이 0주로 떨어지는 경우, 시스템상 진입 불가능한 종목이므로 신호만 기록하고 매수하지 않습니다 (자본 한도 초과 신호).

### 후보 → 신규 편입 절차

A등급 후보를 실제 포트폴리오에 편입하려면:

1. 자본 한도 확인 (1유닛 ≥ 1주 가능한가)
2. 기존 종목 청산 또는 자본 증액으로 슬롯 확보
3. `holdings.json` 에 매수 정보 추가 후 다음 실행부터 추적 시작
4. 동시에 `src/config.ts` 의 STOCKS 배열에도 등록(원하면)

## ✅ 액션 추적 프로토콜

시스템 신호 → 실제 매매 실행 → 추적이 한 화면에서 이루어집니다. 각 종목 카드에 시스템이 권하는 액션(매수/매도)이 표시되고, 사용자는 실제 매매 후 "실행함" 버튼을 눌러 기록합니다.

### 노티 대상 액션

다음 시그널이 발생한 종목에 한해 카드 아래 노티 바가 표시됩니다.

- **매수 노티** (초록):
  - `ENTRY_BREAKOUT` — 신규 매수 (20일 신고가 돌파)
  - `PYRAMID` — 추가 매수 (피라미딩, +0.5 ATR 도달)
- **매도 노티** (주황):
  - `EXIT_10D_LOW` — 익절 청산 (10일 저점 이탈)
  - `STOP_LOSS` — 손절 (-2 ATR)
- 매수 후보 스캔 A등급도 동일한 매수 노티 표시 (실제 진입 시 추적용)

`HOLD`, `WAIT` 는 노티 표시 없음 (조용한 상태).

### 실행 추적

1. 매매 실행 후 "**실행함**" 버튼 클릭
2. 버튼이 "✓ 실행함 (MM.DD HH:MM)" 배지로 전환되고, 카드가 옅은 톤으로 변경
3. 옆에 ↺ 버튼으로 이행 취소 가능 (재진입·중복 등 잘못 누른 경우)
4. 상태는 브라우저 `localStorage` 에 보관되어 페이지 새로고침·재배포에도 유지

### 추적 식별자

키 형식: `turtle-action:{종목코드}:{시그널종류}`

- 같은 종목 + 같은 시그널 = 한 번만 마크
- 매수 사이클(매수 → 매도) 완료 후 다음 매수 시그널이 나오면 동일 키가 이미 "실행함" 상태일 수 있음 → ↺ 로 리셋한 후 새 사이클 추적
- 다른 종목 / 다른 시그널 종류는 독립적으로 추적

## ⏱️ 장중 트리거 실시간 알림 (Phase 2)

일봉 종가 기반 알림 (16:00 KST 발송) 외에, **장중에 핵심 가격대를 통과하는 순간** 잔디로 즉시 알림을 보냅니다.

### 모니터링 주기

- GitHub Actions cron `*/5 0-6 * * 1-5` (평일 KST 09:00~15:30, **5분 간격**)
- 실제 발송 시점은 cron 지연 5~15분 포함

### 감지 트리거

- `STOP_BREACH` — 보유 종목 현재가 ≤ 매수가 − 2 ATR (손절선 이탈)
- `EXIT_10D_BREACH` — 보유 종목 현재가 ≤ 10일 저점 (익절 청산 신호)
- `BUY_TRIGGER_HIT` — 매수 후보 B 등급 현재가 ≥ 20일 고점 (신규 매수 트리거)
- `BREAKOUT_LOST` — 매수 후보 A 등급 현재가 < 20일 고점 (일시 후퇴)

### 중복 방지

- 같은 거래일 + 같은 (종목, 트리거 종류) = **단 1회 발송**
- 캐시 파일: `docs/intraday-alerts.json` (커밋되어 cron 간 공유)
- 자정(KST) 지나면 자동 리셋 (`date` 필드로 판별)

### 신호 vs 알림의 관계

- **신호 (Signal)**: 일봉 종가 기준, 16:00 / 08:30 KST 발송 — 진입·청산 의사결정
- **알림 (Intraday Alert)**: 장중 5분 단위, 가격이 핵심선 통과 시 — 실행 타이밍 캐치

알림이 도착하면 → 시스템대로 매매 → 카드의 "실행함" 버튼 클릭 → 다음 사이클 대기.
